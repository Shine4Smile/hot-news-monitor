# 热点监控系统 — 需求文档

## 项目背景

作为一个关注热点的程序员，需要第一时间获取热点信息（如 AI 大模型更新、时政新闻等），不依赖人工手动搜索，而是利用工具自动发现指定热点变化并及时通知。

项目定位：轻量级工具类项目，敏捷开发，不过多人工介入和工程化。

## 核心功能

### F1: 关键词监控 + AI 验真

- 用户手动输入要监控的关键词（如 "GPT-5"、"DeepSeek"）
- 系统定时从多个信息源采集相关内容
- **利用 DeepSeek AI 识别假冒/不相关内容**，仅保留真实匹配的结果
- 发现匹配内容后第一时间推送通知

### F2: 热点自动发现

- 用户指定关注范围（如 "AI 编程"、"前端技术"）
- 系统每隔一段时间自动搜集该范围内的热点
- 热点以列表/看板形式展示，用户可随时查看

### F3: 通知推送

- 发现匹配关键词的内容时即时通知
- 支持浏览器通知 + 应用内通知中心
- 通知可标记已读

## 产品形态

### Web 应用（优先）

- 响应式兼容的 Web 页面（桌面 + 移动端）
- 独特 UI 设计，拒绝千篇一律
- 功能完整可用

### Agent Skills（第二阶段）

- 封装成 Agent Skills 技能包
- 可交给其他 AI 来监控和发现热点
- 提供标准化的调用接口

## 信息源策略

- 从多个信息源获取，避免单一来源
- 通过后端网页搜索爬虫获取（注意频率控制）
- Twitter (X) 已接入但暂时停用，可随时恢复
- 不做单一 API 依赖

## AI 服务

- 使用 DeepSeek API 原生接口（fetch），零 SDK 依赖
- 用于：关键词扩展(Query Expansion)、内容真实性验证、相关性评分、重要程度评估、内容摘要生成
